## Backend 💽

**Мова програмування:** Golang 🐹

**Використані технології 💻**
- net/http - апі
- gorilla/websocket - вебсокет
- pgx (PostgreSQL) - база даних
- Docker - докер
- testing + testify - тести
- zap - логування

**Запуск ▶️**

- ```docker-compose up --build```
- ```docker-compose logs -f server```

Сервер буде обробляти запити за шляхом - *localhost:8080*

**Функціонал 🪼**

Написавши грамотний frontend до цього серверу, можна отримати проєкт з можливістю створити аккаунт користувача і авторизуватись в нього за допомогою токена, який дійсний 7 днів з моменту створення, або повторного логіну. 
Проєкт зможе стати майданчиком для гри користувачів у хрестики-нолики один проти одного.
Кожен користувач буде мати змогу подивитись таблицю лідерів по рейтингу.

**Мінуси серверу ➖**
- Передбачено, що frontend буде зберігати токен сессії у localstorage, це може бути небезпечним для акаунту користувача.
- Не ідеальний розподіл абстракцій по пакетах. Пакет logic містить в собі http, service і database логіку (хоч і у різних файлах).
- Код може виглядати громіздким.

**Отриманий досвід 💡**

Я:
- Значно просунувся у мові програмування Golang і backend розробці.
- Навчився розділяти абстракції на зони (у вигляді файлів), створив не погану архітектуру.
- Отримав перший досвід роботи в команді.
- Вперше попрацював з Docker.
